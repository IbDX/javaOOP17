Question 1 [MCQ]
Which of the following statements accurately describes the primary difference between text I/O and binary I/O in Java?
Answer: (A) Text I/O involves character encoding and decoding, while binary I/O processes raw binary values.
Explanation: Text I/O involves character encoding and decoding, treating data as sequences of characters. Binary I/O, on the other hand, processes data as raw binary values (sequences of bits) without character conversion, making it generally more efficient for non-textual data.
Question 2 [MCQ]
In Java, what is the primary purpose of the Scanner and PrintWriter classes?
Answer: (B) To handle text-based input and output operations.
Explanation: The Scanner class is used for reading text data from an input source, and the PrintWriter class is used for writing text data to an output destination. They are fundamental for text-based I/O operations.
Question 3 [MCQ]
Consider the decimal integer 199. How is it typically stored in a text file compared to a binary file, according to Java I/O principles?
Answer: (C) As a sequence of characters '199' in a text file and as a byte value 0xC7 in a binary file.
Explanation: In a text file, the decimal integer 199 would be stored as a sequence of three characters ('1', '9', '9'), each represented by its character encoding (e.g., ASCII). In a binary file, it would be stored as its byte-type value, which is 0xC7 (hexadecimal) or 199 (decimal) directly.
Question 4 [MCQ]
Which of the following statements is true regarding the efficiency and portability of binary I/O compared to text I/O?
Answer: (A) Binary I/O is more efficient and portable than text I/O.
Explanation: Binary I/O is generally more efficient because it does not require character encoding and decoding. It is also more portable because binary files are independent of the host machine's encoding scheme, allowing Java programs to read them on any machine.

Question 8 [MCQ]
Why do most methods in Java's binary I/O classes declare to throw java.io.IOException or a subclass of it?
Answer: (A) To ensure that potential I/O errors are explicitly handled by the programmer.
Explanation: I/O operations are prone to various errors, such as file not found, permission issues, or disk full. IOException is a checked exception, meaning the compiler forces the programmer to either handle it (using a try-catch block) or declare that the method throws it, ensuring robust error handling.

Question 11 [MCQ]

Question 12 [MCQ]
Which of the following statements correctly describes the function of DataInputStream and DataOutputStream?
